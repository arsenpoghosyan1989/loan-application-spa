# Loan Application SPA

Одностраничное приложение с тремя последовательными формами для оформления заявки на займ. Данные из всех шагов сохраняются в общем состоянии и доступны на любом шаге; финальная модалка показывается после успешного POST-запроса на тестовый API.

## Стек и обоснование выбора библиотек

| Библиотека | Зачем |
|---|---|
| **Vite 6 + React 19 + TypeScript 5.7** | Актуальная версия React, быстрый dev-сервер и сборка. |
| **React Router v7** | Требование ТЗ — переключение между формами через маршруты `/step-1`, `/step-2`, `/step-3`. |
| **React Bootstrap 2 + Bootstrap 5** | ТЗ допускает Bootstrap; React Bootstrap даёт типизированные компоненты (`Form`, `Modal`, `Button`). |
| **React Hook Form** | Производительная работа с формами: неконтролируемые инпуты, минимум ре-рендеров. |
| **Zod + @hookform/resolvers** | Декларативная схема валидации + автоматический вывод TypeScript-типа (`z.infer`). Один источник правды для типов и проверок. |
| **react-imask** | Маска телефона `0XXX XXX XXX` — готовое решение, без написания своего парсера. |
| **TanStack Query (React Query)** | Кэш списка категорий на уровне приложения. При возврате со step-3 на step-2 запрос к `dummyjson` не повторяется (staleTime = 10 мин) — это и есть «переиспользование результата», требуемое в ТЗ. Также используется для мутации `POST /products/add` с `isPending`/`isError`. |

Обоснование выбора также дублируется комментариями в коде (см. `main.tsx`, `Step1Personal.tsx`, `hooks/useCategories.ts`, `api/products.ts`).

## Запуск

Требования: **Node.js 20.19+** или **22.12+**, **pnpm 10+**.

```bash
pnpm install
pnpm dev          # запускает dev-сервер на http://localhost:5173
```

### Другие команды

```bash
pnpm build        # сборка production (с проверкой типов)
pnpm preview      # локальный предпросмотр production-сборки
pnpm lint         # (опционально, если настроите ESLint)
```

## Как проверить работоспособность

1. **Форма 1:** введите телефон (маска `0XXX XXX XXX`), имя, фамилию, выберите пол. Попробуйте оставить поле пустым и нажать «Далее» — должна показаться ошибка.
2. **Форма 2:** подождите загрузку списка категорий (запрашивается один раз и кэшируется). Выберите место работы, введите адрес. Проверьте кнопку «Назад» — данные с шага 1 должны сохраниться.
3. **Форма 3:** подвигайте слайдеры (сумма $200–$1000 с шагом $100, срок 10–30 дней). Нажмите «Подать заявку» — должен уйти POST-запрос на `https://dummyjson.com/products/add` с телом `{"title": "Имя Фамилия"}` и появиться модалка с текстом:

   > Поздравляем, `<Фамилия> <Имя>`. Вам одобрена `<сумма>` на `<срок>` дней.

4. **Кэш API:** откройте DevTools → Network, вернитесь со step-3 на step-2 и снова на step-3 — повторного запроса `GET /products/category-list` быть не должно.

## Ограничения валидации

- Телефон обязателен, должен соответствовать маске `0XXX XXX XXX`.
- Имя/фамилия/адрес — обязательные непустые строки (trim).
- Пол, место работы — обязательный выбор из списка.
- Сумма и срок — гарантированы диапазонами range-инпутов.
